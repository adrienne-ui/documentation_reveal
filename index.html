Segment Lab — User Guide

URL: https://segment-lab.qwarry.co/ · Internal platform · Last updated: May 2026

1. Introduction

Segment Lab is Qwarry's internal segment management platform. It is the central tool used by AdOps and Sales teams to create, configure, and activate audience segments across Qwarry's partner ecosystem (Xandr, Index Exchange, Google).

Segment Lab allows you to:

  • Create and manage all types of targeting segments (contextual, YouTube, socio-demo, technical, language, combinations)
  • Activate segments on DSP/SSP partners via partner mappings
  • Monitor reach — view impressions and user counts per segment
  • Tune YouTube targeting — use AI-powered vector search to find relevant channels and videos
  • Track commercial usage — link segments to Zoho CRM deals
Who uses Segment Lab?
• AdOps — Create/edit segments, tune YouTube targeting, manage partner mappings
• Sales — View available segments, check reach data, link segments to deals
• Customers — Request custom segments via their account manager

The Segment Lifecycle

  1. Creation — A segment is created in Segment Lab with its parameters (name, description, rules)
  2. Configuration — Thresholds, keywords, or conditions are set depending on segment type
  3. Mapping — The segment is mapped to one or more partners (Xandr, Index Exchange)
  4. Activation — The segment becomes available for targeting on the partner platform
  5. Monitoring — Reach data (impressions, users) is collected and displayed

4. Measured Categories

Measured Categories are the building blocks of all segments. They are the 526 semantic categories scored by Qwarry's technology. Each category represents a topic (e.g. "Automotive", "Food & Drink", "Personal Finance") and has:

  • Name — e.g. "Food & Drink", "Automotive", "Personal Finance"
  • Metadata per language — Description, keywords, and exclusion list for each supported language
  • Quality metrics — Accuracy, precision, and recall scores
URL Samples: You can view sample URLs that match a measured category with a given threshold. This helps validate that the category is correctly identifying relevant content before creating a SEMO.

5. SEMOs (Semantic Models)

A SEMO is the most fundamental segment type. It targets web pages whose content matches a specific semantic category above a defined threshold.

What Is a SEMO?

When Qwarry's technology analyzes a web page, it assigns scores to hundreds of categories (called Measured Categories). A SEMO defines a rule: "target pages where category X scores above threshold Y".

Web Page → Semantic Scoring
(400+ categories) → SEMO Rule
(category + threshold) → Page qualifies
for targeting

6. Contextual Audiences

Contextual Audiences target users (IDs) rather than pages. They are built from users who have visited pages categorized by Qwarry's technology.

How Contextual Audiences Differ from SEMOs

SEMOContextual Audience
TargetsA web pageA user (ID)
Logic"This page is about X""This user has visited pages about X"
ReachOnly on matching pagesAnywhere the user browses
SitelistCompatibleNot recommended (limits reach)

Audience Parameters

ParameterDescriptionExample
NameAudience name"Eco-Conscious Shoppers FR"
Affiliated SegmentsSEMOs that define the audience (1 to 100+)["Organic Food", "Sustainable Living"]
FrequencyMinimum visits to qualify3 (user must visit 3 matching pages)
RecencyTime window (max 90 days)30 days
CountriesGeographic scope["FR", "DE", "ES"]
Audience Typecontextual or inappcontextual
Statusactive, inactive, or archivedactive

Frequency & Recency Tuning

  • Higher frequency = more engaged users, smaller but more qualified audience
  • Lower frequency = broader reach, less qualified
  • Shorter recency = fresher intent (user visited recently)
  • Longer recency = larger pool but older interest
Important: Recency cannot exceed 90 days due to data retention limits. Audiences are recalculated daily based on the configured parameters.

The Audiences list shows all 260 contextual audiences. Each card displays the ID, name, audience code, number of affiliated segments (e.g. "31 segments"), and country (e.g. "FR"). Two actions are available: "Edit" to modify the audience, and "Archive" (pink button) to deactivate it. The search bar supports filtering by ID, audience code, or name.

The audience detail view has 3 tabs: Infos, Stats, and Mapping. The header shows the audience name, an "Active" status badge (green), and a toggle switch to activate/deactivate the audience. The Infos tab displays:

  • Audience code — the unique identifier used by partners
  • Countries — geographic scope (shown as badges, e.g. "France FR")
  • Affiliated segments — all the SEMOs that define this audience (displayed as grey badges). Users who visit pages matching any of these segments will be included in the audience
  • Frequency — minimum number of visits required (e.g. "1 hit")
  • Recency — time window for counting visits (e.g. "30 days")

7. Combination Segments

Combination Segments allow you to create complex targeting rules by combining multiple measured categories with AND/OR logic.

How Combinations Work

A Combination Segment uses a nested logical structure:

AND
(all must match) contains OR groups
(at least one must match) contains Conditions
(category + threshold + operator)

Example

Target pages that are about Automotive AND (Luxury OR Premium):

LevelLogicCondition
ANDAll OR groups must match—
→ OR Group 1At least one conditionAutomotive > 70
→ OR Group 2At least one conditionLuxury > 60 OR Premium > 60

Combination Parameters

ParameterDescription
NameDescriptive name
DescriptionExplanation of the targeting logic
CodeAuto-generated unique code for partner activation
AND conditionsNested OR groups, each containing category/threshold/operator conditions
Use case: Combinations are ideal for advertisers who need precise contextual targeting — e.g. "pages about Travel AND Luxury" or "pages about Health AND NOT Tobacco".

The list displays all 232 combination segments. Each card shows the ID, name (e.g. "284 – The Fashion Lover – Standard Brand Safety"), and the segment code. Click "Edit" to modify or "+ New" to create a new combination.

The detail view has 2 tabs: Infos and Mapping. The Infos tab shows the segment code and the Expression — the logical formula that defines the combination. The expression uses a compact syntax: ((2281>=20)|(2417>0))&((1615<400))&... where numbers are measured category IDs, operators define thresholds, & means AND, and | means OR. This allows very precise multi-condition targeting.

The Tuning Process

YouTube Segments are created and refined through a tuning process:

  1. Define the segment — Provide a name, description, keywords, and exclusion keywords
  2. AI Enhancement — Optionally use AI to generate better descriptions, keywords, or exclusions
  3. Run tuning — The system searches for matching channels across 3 tiers (Large, Medium, Small)
  4. Review results — Inspect the matched channels and videos
  5. Adjust parameters — Fine-tune thresholds, number of results, and exclusions
  6. Save — Confirm the segment, which triggers daily automatic synchronization

Channel Tiers

Channels are categorized by subscriber count into three tiers:

TierSubscribersUse Case

Medium50,000 – 500,000Balanced reach and relevance
Small< 50,000Niche, highly relevant channels

AI Enhancement

Segment Lab can use AI (AWS Bedrock Nova) to automatically improve your segment definition:

  • Generate description — Create a rich semantic description from a segment name
  • Suggest keywords — Propose relevant keywords to improve search recall
  • Suggest exclusions — Identify keywords that could bring irrelevant results

Daily Synchronization

Once a YouTube Segment is saved, it is automatically refreshed daily:

  • New channels matching the criteria are added
  • Channels that no longer match are removed
  • Videos from matched channels are indexed

Languages are processed in rotation (e.g. FR+EN on day 1, IT+DE on day 2, etc.) to distribute load.

The Tuning Interface

The tuning page is the core interface for configuring YouTube segment search. The left panel contains the form fields:

  • Name — segment name (read-only during tuning)
  • Description* — the semantic description that gets converted into an embedding vector
  • Keywords* — additional terms to improve search recall (add with Enter key, remove with ✕)
  • Exclusion keywords — terms to filter out (optional)

The right panel shows Recommendations with suggested parameter values for different use cases:

  • Broad theme (entertainment, fashion, humor) — Min OpenSearch score: 0.60, Scan K: 2000, keep standard quotas
  • Niche theme (wine, finance, specific sport) — Min OpenSearch score: 0.50, Scan K: 3000, reduce Large channels, increase Small channels

Tips at the bottom: "Not enough results? → Lower Min OpenSearch score or Min Reranking score" and "Too many off-topic results? → Increase Min Reranking score to 0.25–0.30".

Three action buttons: "Reset" (pink, clears changes), "Preview" (runs tuning without saving), and "Save" (green, runs tuning and saves results).

When tuning is launched (Preview or Save), the right panel switches to a Steps view showing real-time progress. Each step displays its name, duration, and a green checkmark when complete:

  1. Generating embedding (0.7s) — converts description + keywords into a vector
  2. Searching large channels (2.5s) — k-NN search in the Large tier (Found: 60 → Filtered: 60)
  3. Reranking large channels (0.5s) — AI reranking reduces results (Before: 60 → After: 4)
  4. Searching medium channels — currently in progress (animated spinner)

The form fields become greyed out (disabled) during tuning. A "Cancel" button (red) allows stopping the process.

After tuning completes, the results are displayed in columns by tier:

  • Large channels (4) — e.g. Don Omar (12.3M subs, US), abdoulupnext (2.4M subs, US)
  • Medium channels (2) — e.g. Kryssy (79K subs, FR), Chabine Prod Urban Channel (191K subs, FR)
  • Small channels (11) — e.g. CollectifMetisseVEVO (35.8K subs), DJ Max's (2.5K subs, FR)
  • Videos (1) — individual videos matched by content

Each result shows the channel avatar, name, subscriber count, country, and a relevance score (in green, e.g. "Score: 0.26"). The left panel shows the complete steps log with a "Collapse" button. Buttons at the bottom: "Reset", "Preview", and "Save".

11. In-URL Keywords Segments

In-URL Keywords segments target pages where specific keywords appear in the page URL. This is a simple but effective targeting method for pages with descriptive URLs.

How It Works

The system checks if any of the defined keywords are present in the URL string of a web page. If a match is found, the page qualifies for the segment.

Example

Segment: "Electric Vehicles" with keywords: ["electric-car", "ev-", "tesla", "electric-vehicle"]

Matching URLs:

  • https://news.com/2026/05/electric-car-sales-record ✓
  • https://auto.fr/reviews/tesla-model-3 ✓
  • https://finance.com/stocks/apple-earnings ✗

Parameters

ParameterDescription
NameSegment name
DescriptionWhat this segment targets
KeywordsArray of keywords to match in URLs
CodeAuto-generated unique code
Best practice: Use URL-friendly keywords (hyphenated, lowercase). Avoid overly generic terms that would match too many pages.

The list displays all 118 In-URL Keywords segments. Each card shows the ID, name (e.g. "333 – Purites", "267 – Specific – Danone – IFFO"), and the segment code. Click "Edit" to modify or "+ New" to create a new segment.

The detail view has 2 tabs: Infos and Mapping. The Infos tab shows the segment code and the Keywords — the list of terms that are matched against page URLs. Keywords are displayed as a continuous text string (e.g. "daily hygiene routine feminine hygiene daily stay fresh all day personal care routine..."). Any page whose URL contains one of these terms will qualify for the segment.

The Mapping tab shows which partners this segment is activated on. In this example, segment #113 is mapped to Xandr (ID: 6), created by tech@qwarry.com on 02/10/2024. The "Edit" button allows adding or removing partner mappings.

Important: When you map a segment to a partner, it is automatically registered on the partner's platform via their API. Removing a mapping deactivates the segment on that partner.

17. Zoho Deals Integration

Segment Lab integrates with Zoho CRM to link segments to commercial deals, providing visibility on which segments are used in active campaigns.

How It Works

  • Zoho deals are synchronized every hour automatically
  • Each deal can reference multiple segment types: Contextual Audiences, SEMOs, Custom Tribes, Socio-Demo
